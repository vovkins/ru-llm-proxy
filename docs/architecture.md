# Архитектура ru-llm-proxy

## Обзор

```
┌──────────┐     ┌──────────────┐     ┌─────────────┐     ┌──────────────┐
│  Клиент   │────▶│ LiteLLM Proxy│────▶│   Presidio   │────▶│  LLM Provider│
│ (приложение)│   │  (порт 4000) │     │  Analyzer    │     │ (OpenAI, etc)│
└──────────┘     └──────────────┘     └─────────────┘     └──────────────┘
                        │                     │
                        │              ┌─────────────┐
                        │              │ Anonymizer   │
                        │              │ (маскировка) │
                        │              └─────────────┘
                        │
                 ┌──────┴──────┐
                 │ PostgreSQL  │
                 │   + Redis   │
                 └─────────────┘
```

## Поток запроса

1. Клиент отправляет запрос к LiteLLM API (OpenAI-совместимый формат)
2. LiteLLM передаёт текст через Presidio guardrail (pre-call)
3. Presidio Analyzer обнаруживает PII (имена, телефоны, ИНН и т.д.)
4. Presidio Anonymizer заменяет PII на плейсхолдеры: `[ИМЯ_1]`, `[ТЕЛЕФОН_1]`
5. Обезличенный запрос отправляется к LLM-провайдеру
6. Ответ от LLM проходит через Presidio Anonymizer (post-call)
7. Плейсхолдеры в ответе заменяются на оригинальные данные
8. Клиент получает полный ответ с оригинальными данными

## PII-детекция для русского языка

### Regex-recognizers (точные паттерны)
- Российские телефоны: +7/8, международные форматы
- Email-адреса
- ИНН физических и юридических лиц (с контрольной суммой)
- СНИЛС (с проверкой)
- Паспортные данные (серия/номер)
- Номера банковских карт (Luhn-проверка)
- Базовые паттерны адресов

### NER-модель (DeepPavlov ruBERT)
- Имена людей (ФИО)
- Адреса (улица, дом, квартира)
- Названия организаций

## Стек версий

| Компонент | Версия | Примечание |
|-----------|--------|------------|
| Python | 3.12 | Для Presidio recognizers и тестов |
| LiteLLM | 1.86+ (latest stable) | Docker-образ `ghcr.io/berriai/litellm:main-latest` |
| Presidio Analyzer | 2.2.362+ | `pip install presidio-analyzer` |
| Presidio Anonymizer | 2.2.362+ | `pip install presidio-anonymizer` |
| spaCy | 3.8+ | NLP-движок для Presidio |
| Natasha/Slovnet | latest | Лёгкая альтернатива DeepPavlov для NER (~30 MB, на 1-2% хуже BERT SOTA, но в 60 раз меньше) |
| DeepPavlov | 1.0+ | NER-модель ruBERT (полная точность, ~2 GB) |
| PostgreSQL | 16-alpine | БД для LiteLLM |
| Redis | 7-alpine | Кэширование, сессии |
| Docker | 20.10+ | Контейнеризация |
| Docker Compose | v2 | Оркестрация |

**Примечание по NER:** Natasha/Slovnet — более лёгкий вариант (30 MB vs 2 GB), качество NER на 1-2% ниже DeepPavlov ruBERT, но значительно быстрее и экономичнее. Для MVP можно начать с Natasha, при необходимости перейти на DeepPavlov.
