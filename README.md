# ru-llm-proxy

LLM-прокси с санитайзером персональных данных (PII) для русского языка.

Обнаруживает и маскирует чувствительные данные (имена, телефоны, адреса, ИНН, СНИЛС, паспортные данные, номера карт) в запросах к LLM перед отправкой провайдеру, и восстанавливает оригинальные данные в ответах.

## Архитектура

- **LiteLLM Proxy** — единый OpenAI-совместимый API для 100+ LLM-провайдеров
- **Microsoft Presidio** — фреймворк для обнаружения и маскирования PII
- **Русские recognizers** — кастомные детекторы для российских форматов данных
- **DeepPavlov NER (ruBERT)** — нейросетевая модель для распознавания имён, адресов, организаций

## Компоненты

| Компонент | Назначение |
|-----------|------------|
| LiteLLM | Агрегация LLM-провайдеров, роутинг, guardrails |
| Presidio Analyzer | Детекция PII в тексте |
| Presidio Anonymizer | Маскирование и восстановление PII |
| PostgreSQL | Хранение конфигурации LiteLLM, ключей, логов |
| Redis | Кэширование, сессии |

## Быстрый старт

```bash
# 1. Клонировать репо
git clone https://github.com/vovkins/ru-llm-proxy.git
cd ru-llm-proxy

# 2. Создать .env из шаблона и заполнить ключи
cp .env.example .env

# 3. Запустить
make setup  # первичная настройка (генерация ключей, скачивание моделей)
make up     # запуск всех сервисов
```

## Конфигурация

- `litellm-config.yaml` — настройки LiteLLM (провайдеры, роутинг, guardrails). Монтируется через volume — можно менять без пересборки
- `.env` — секреты (API-ключи провайдеров, мастер-ключ)

## Make-команды

| Команда | Описание |
|---------|----------|
| `make setup` | Первичная настройка (ключи, модели) |
| `make up` | Запуск всех сервисов |
| `make down` | Остановка |
| `make restart` | Рестарт с применением нового конфига |
| `make logs` | Логи всех сервисов |
| `make test` | Запуск тестов recognizers |
